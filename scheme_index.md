# scheme_index.md — 首頁與書卷目錄撰寫指南

本文說明 `index.md`（全庫首頁）與各書卷 `全書目錄及綱要.md` 的撰寫與維護規則。

---

## 1. index.md — 全庫首頁

### 位置

```
scripture/index.md
```

### 用途

給一般讀者看的入口頁。不應放入開發細節、驗證標準或工具指令。

### 應包含

| 區塊 | 說明 |
|------|------|
| 專案簡介 | 一句話說明這知識庫是什麼（經文＋註解＋主題） |
| 按書卷研讀 | 已完成書卷的表格式目錄，每卷用 `[[書卷資料夾/全書目錄及綱要\|書卷名]]` 連結 |
| 按主題查詢 | 條目列表，分神學主題／核心人物／關鍵地點／原文與文化 |
| 資料來源 | CT/GT/KC/BH 簡介表 |
| 使用方式 | 3-4 條讀者使用指引 |
| 相關文件 | README、INSTALL_COMPUTER、INSTALL_MOBILE 連結 |
| 尾行 | 最新完成進度統計 |

### 不應放入

- 未完成書卷的完整進度表（用「建置中」概括即可）
- Mermaid 流程圖、驗證指令、git 指令
- 開發用目錄結構（raw_data/、util/ 等）

### 更新時機

- 每完成一卷書時，在「按書卷研讀」加入該卷連結
- 每完成一章時，更新尾行進度數字
- 有新的熱門主題條目時，可加入「按主題查詢」

### 連結格式

- 書卷連結：`[[01 創世記/全書目錄及綱要\|創世記]]`（顯示書卷名，導向該卷目錄）
- 主題條目：`[[條目完整名稱]]` 或 `[[條目完整名稱\|顯示縮寫]]`

## 2. 全書目錄及綱要.md — 各卷書目錄

### 位置

```
scripture/《書卷資料夾》/全書目錄及綱要.md
```

例：
```
scripture/01 創世記/全書目錄及綱要.md
scripture/02 出埃及記/全書大綱.md     ← 檔名可自行命名
```

### 用途

給讀者快速定位該書卷的章節。從 `index.md` 的書卷連結點進來，就是在這個頁面。

### 基本結構

```markdown
## 🗂️ 《書卷名》目錄

### 第一階段名稱（章節範圍）
[[0X 《書卷名》/第x章.md|第x章|]] · [[0X 《書卷名》/第x+1章.md|第x+1章|]] · ...

### 第二階段名稱（章節範圍）
...
```


### 設計原則

1. **依書卷客製化分階段**：每卷書的文學結構不同，分階段應反映該卷本身的敍事轉折。例如創世記分四個人物時代（始祖→亞伯拉罕→以撒雅各→約瑟），出埃及記可按「在埃及／出埃及／西乃立約」三分。

2. **每章用完整 wiki-link**：`[[0X 書卷名/第x章|第x章]]`，全章號並排，用 `·` 分隔。用跨資料夾的完整路徑確保無論從哪個頁面點擊都能正確抵達。

3. **各階段加一句話說明**：每個 `###` 階段標題下，可加一句該階段的內容概要，幫助讀者理解這段經文的主軸。

4. **不要在目錄檔放主題條目**：主題條目屬於 `index.md` 或 `link_folder/`。書卷目錄專注於章節導航。

### 對應的目錄檔可包含

| 元素 | 說明 |
|------|------|
| 階段標題 | `### 階段名稱（章節範圍）` |
| 階段概要 | 一句話說明該階段核心事件 |
| 章節連結 | `[[0X 書卷名/第x章\|第x章]]` 以 `·` 串聯 |

### 不應包含

- link_folder 主題條目列表（那是 index.md 的工作）
- 每章的詳細摘要（那是各章主檔的工作，目錄只做階段層級的概要）
- 全庫統計資料

## 3. index.md 與全書目錄及綱要.md 的搭配

```
讀者進入 index.md
  → 看到「按書卷研讀」表，點選 [[01 創世記/全書目錄及綱要|創世記]]
    → 進入 全書目錄及綱要.md
      → 看到四大階段＋50章連結
        → 點 [第42章](第42章.md)
          → 進入第42章研讀頁（經文＋wiki-link＋知識節點＋整理）
            → 點 [[約瑟的夢（應驗）]]
              → 進入該條目頁，看定義＋跨章累積＋來源
```

index.md 是 **入口**，書卷目錄頁是 **該卷的導航中樞**，章節頁是 **研讀終點**。

---

## 4. 補寫流程

### 補寫新書卷目錄

1. 確認該書卷的文學結構（參考查經資料或既有大綱）
2. 將全部章節依結構分為 2-5 個階段，各階段命名反映該段內容
3. 為各階段寫一句概要
4. 替換章節連結，確保指向 `第x章.md`（即使章節尚未建立，連結可預留）
5. 完成後在 `index.md` 的「按書卷研讀」表加入該卷的一列

### 補寫 index.md（首頁新增書卷後）

1. 在「按書卷研讀」表新增該卷列：`[[書卷資料夾/目錄檔名\|書卷名]] | 章數 | 已完成數 | 進度條`
2. 若該卷新增了重要的主題條目，可加入「按主題查詢」
3. 更新尾行進度數字

## 6. 注意事項

- **尚未建立的章節也可預留連結**：Obsidian 中未建立的連結會顯示為淡色，不影響使用。
- **階段概要保持一句話**：詳細內容留給各章主檔。目錄頁只需讓讀者知道這段在講什麼。
- **index.md 不要膨脹**：全庫有 66 卷書，即使只顯示有進度的書卷，表格也不應過長。尚未開工的書卷一律概括處理。
